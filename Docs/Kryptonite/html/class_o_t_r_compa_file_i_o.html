Detailed Description

Helper class dynamically import _wfopen_s and fclose from msvcrt.dll.

Why do so? Because libOTR-2.dll uses fwrite from msvcrt.dll on our handles and fwrite produces an access violation in case the given file handle was created with _wfopen_s from a different runtime lib like e.g. msvcr100.dll.
